# Data Model: Packaging & Distribution Modernization

**Feature**: 002-packaging-modernization | **Date**: 2026-01-11

## Overview

This document defines the data structures and entities involved in the modernized packaging and distribution system.

## Entity Definitions

### 1. Release Artifact

**Description**: Represents a complete release package including installer, portable executable, and metadata.

**Fields**:

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `version` | String (semver) | Required, matches `/^v?\d+\.\d+\.\d+/` | Semantic version (e.g., "0.5.1") |
| `buildDate` | String (ISO 8601) | Required | Build timestamp in ISO 8601 format |
| `commitSHA` | String | Required, 40 chars (SHA-1) | Git commit hash for traceability |
| `nodeVersion` | String | Required | Node.js version used for build (e.g., "20.10.0") |
| `npmVersion` | String | Required | npm version used for build (e.g., "10.2.3") |
| `electronVersion` | String | Required | Electron version (e.g., "33.4.11") |
| `installerPath` | String (file path) | Required | Path to NSIS installer (e.g., "dist/MarkRead-Setup-0.5.1.exe") |
| `portablePath` | String (file path) | Required | Path to portable .exe (e.g., "dist/MarkRead-0.5.1-portable.exe") |
| `blockMapPath` | String (file path) | Required | Path to blockmap for differential updates (e.g., "dist/MarkRead-Setup-0.5.1.exe.blockmap") |
| `updateManifestPath` | String (file path) | Required | Path to latest.yml manifest (e.g., "dist/latest.yml") |

**Relationships**:
- Has one `Code Signing Certificate` (used for signing)
- Published to one `GitHub Release` (distribution platform)
- Generates one `Update Manifest` (for auto-update discovery)

**Validation Rules**:
- `version` must match package.json version (FR-004)
- `commitSHA` must exist in git history
- All file paths must exist before publishing

**State Transitions**:
1. **Created**: Artifacts generated by electron-builder
2. **Signed**: Installer signed with code signing certificate
3. **Verified**: Signature verified with signtool
4. **Published**: Uploaded to GitHub Releases
5. **Failed**: Build or signing failed, artifacts discarded

### 2. Code Signing Certificate

**Description**: Digital certificate used to sign installers for Windows SmartScreen trust.

**Fields**:

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `source` | Enum: `PFX_FILE` \| `CERT_STORE` | Required | Certificate storage location |
| `thumbprint` | String | Required if source=CERT_STORE, 40 chars (SHA-1) | Certificate thumbprint for Windows Certificate Store |
| `pfxPath` | String (file path) | Required if source=PFX_FILE | Path to PFX certificate file |
| `password` | String (secure) | Required if source=PFX_FILE | Password for PFX file (from env variable) |
| `expirationDate` | String (ISO 8601) | Required | Certificate expiration date |
| `subjectName` | String | Required | Certificate subject (e.g., "CN=MarkRead Project") |
| `issuer` | String | Optional | Certificate issuer (CA name) |
| `algorithm` | String | Required, value="SHA256" | Signing algorithm |
| `timestampServer` | String (URL) | Required, default="http://timestamp.digicert.com" | RFC 3161 timestamp server |

**Relationships**:
- Signs multiple `Release Artifacts`

**Validation Rules**:
- `expirationDate` must be > current date (FAIL if expired)
- `expirationDate` > current date + 30 days (WARN if <30 days remaining) (FR-008)
- `algorithm` must be "SHA256" (SHA-1 not accepted)
- Certificate must be valid (signtool verify after signing)

**Lifecycle**:
1. **Active**: More than 30 days until expiration
2. **Expiring Soon**: Less than 30 days until expiration (build warns)
3. **Expired**: Past expiration date (build fails)
4. **Renewed**: New certificate issued, GitHub Secrets updated

### 3. Update Manifest (latest.yml)

**Description**: YAML manifest published to GitHub Releases for auto-update discovery.

**Generated by**: electron-builder publish phase
**Consumed by**: electron-updater in production mode

**Fields**:

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `version` | String (semver) | Required | Release version (e.g., "0.5.1") |
| `releaseDate` | String (ISO 8601) | Required | Release publish timestamp |
| `path` | String | Required | Relative path to installer (e.g., "MarkRead-Setup-0.5.1.exe") |
| `sha512` | String | Required, 128 chars (base64) | SHA512 checksum for integrity verification |
| `size` | Number | Required | File size in bytes |
| `githubArtifactName` | String | Optional | GitHub Release asset name |
| `releaseNotes` | String (Markdown) | Optional | Extracted from CHANGELOG.md |

**Example**:
```yaml
version: 0.5.1
releaseDate: '2026-01-11T10:30:00.000Z'
path: MarkRead-Setup-0.5.1.exe
sha512: YXNkZmFzZGZhc2Rm... (base64)
size: 87654321
githubArtifactName: MarkRead-Setup-0.5.1.exe
releaseNotes: |
  ## What's New
  - Modernized packaging infrastructure
  - Auto-update support
  - Portable .exe variant
```

**Relationships**:
- Generated from one `Release Artifact`
- Published to one `GitHub Release`
- Consumed by electron-updater for version checks (FR-009)

**Validation Rules**:
- `sha512` must match actual file checksum
- `size` must match actual file size
- `path` must point to valid installer

### 4. Installation Log Entry

**Description**: Structured log entry for installation events, errors, and diagnostics.

**Storage Location**: `%LOCALAPPDATA%\MarkRead\Logs\install-{timestamp}.log`

**Fields**:

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `timestamp` | String (ISO 8601) | Required | Event timestamp with timezone |
| `level` | Enum: `INFO` \| `WARN` \| `ERROR` | Required | Log severity level |
| `errorCode` | String | Required for ERROR level | Machine-readable error code (e.g., "DISK_SPACE_INSUFFICIENT") |
| `message` | String | Required | Human-readable error message |
| `context` | Object | Optional | Additional context (file paths, sizes, etc.) |
| `stackTrace` | String | Optional | JavaScript stack trace for exceptions |

**Standard Error Codes**:

| Error Code | Description | User Action |
|------------|-------------|-------------|
| `DISK_SPACE_INSUFFICIENT` | Not enough disk space | Free up disk space and retry |
| `PERMISSION_DENIED` | Insufficient permissions | Run as administrator or choose per-user install |
| `CERT_EXPIRED` | Code signing certificate expired | Contact developer for updated installer |
| `DOWNLOAD_FAILED` | Update download failed | Check internet connection and retry |
| `UPDATE_ROLLBACK` | Update failed, rolled back | Check logs, report issue to developer |
| `SIGNATURE_INVALID` | Installer signature verification failed | Re-download installer from official source |
| `VERSION_MISMATCH` | package.json version != git tag | Internal build error, report to developer |

**Example Log Entry**:
```json
{
  "timestamp": "2026-01-11T10:30:45.123Z",
  "level": "ERROR",
  "errorCode": "DISK_SPACE_INSUFFICIENT",
  "message": "Installation requires 100MB of free space, but only 50MB available",
  "context": {
    "requiredSpace": 104857600,
    "availableSpace": 52428800,
    "installPath": "C:\\Users\\Alice\\AppData\\Local\\Programs\\MarkRead"
  }
}
```

**Relationships**:
- Generated during `Release Artifact` installation
- Persists beyond application uninstallation
- Referenced in user-facing error dialogs (FR-015)

**Log Rotation**:
- Keep: Last 10 log files
- Delete: Older logs automatically on app startup
- Max size: No limit (installations are fast, logs are small)

### 5. Portable Settings

**Description**: Configuration for portable .exe variant behavior.

**Storage Location**: Relative to .exe (e.g., `./portable-data/settings.json`)

**Fields**:

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `isPortable` | Boolean | Required, value=true | Indicates portable mode |
| `dataDirectory` | String (relative path) | Required, value="./portable-data" | Local data storage directory |
| `disabledFeatures` | Array<String> | Required | Features disabled in portable mode |
| `appSettings` | Object | Optional | User preferences (theme, font size, etc.) |

**Disabled Features in Portable Mode**:
- Auto-updates (`electron-updater` disabled)
- File associations (no registry writes)
- Context menu integration (no registry writes)
- Start menu shortcuts (no installation)
- Desktop shortcuts (no installation)

**Behavior Differences**:

| Feature | Installed Version | Portable Version |
|---------|------------------|------------------|
| Installation | Required | No installation |
| Registry writes | Yes (file associations) | No (zero system modifications) |
| Auto-updates | Yes (FR-009) | No (FR-006) |
| Settings location | %LOCALAPPDATA%\MarkRead | ./portable-data |
| Shortcuts | Desktop + Start Menu | None |
| Uninstallation | Required | Just delete .exe |

**Detection in Code**:
```typescript
const isPortable = !!process.env.PORTABLE_EXECUTABLE_FILE;
```

**Relationships**:
- Variant of `Release Artifact`
- Uses different `Storage Location` than installed version

## Data Flow Diagrams

### Build and Release Flow

```
1. Developer: git push tag v0.5.1
2. GitHub Actions: Trigger release.yml workflow
3. Release Workflow:
   a. Validate version (package.json == tag)
   b. Run tests, lint, type-check
   c. Build application (npm run build)
   d. Package with electron-builder
      → Creates Release Artifact (installer + portable + blockmap + latest.yml)
   e. Load Code Signing Certificate (from GitHub Secrets)
   f. Check certificate expiration (WARN if <30 days)
   g. Sign installer with signtool
   h. Verify signature
   i. Generate build-metadata.json
   j. Create GitHub Release
   k. Upload all artifacts
4. GitHub Releases: Publish release
5. electron-updater: Discover new version via latest.yml
```

### Auto-Update Flow

```
1. User: Launch MarkRead
2. App: Wait 5 seconds (non-blocking)
3. electron-updater: Check latest.yml from GitHub Releases
4. Decision:
   a. If update available:
      → Download installer in background
      → Show notification to user
      → User chooses: Install now | Install on quit | Dismiss
   b. If update unavailable:
      → Schedule next check (4 hours)
   c. If GitHub API fails:
      → Log error silently
      → Retry with exponential backoff (1hr, 2hr, 4hr)
      → Resume normal 4hr interval after success
5. If update installed and crashes:
   → Auto-rollback to previous version backup
   → Log failure to Installation Log
   → Show notification about rollback
```

### Portable .exe Usage Flow

```
1. User: Download MarkRead-0.5.1-portable.exe
2. User: Run .exe from any location (USB drive, Downloads, etc.)
3. App: Detect portable mode (process.env.PORTABLE_EXECUTABLE_FILE)
4. App: Initialize with portable settings:
   a. Data directory: ./portable-data (relative to .exe)
   b. Disable auto-updates
   c. Skip file association checks
   d. Skip registry writes
5. User: Uses app normally (full functionality)
6. App: Saves all settings to ./portable-data/settings.json
7. User: Closes app (no system modifications)
8. User: Delete .exe to "uninstall" (no traces left)
```

## Database Schema

**N/A** - This feature does not require a database. All data is stored as:
- Files: Installers, logs, portable settings
- GitHub Releases: Distribution artifacts
- Windows Registry: File associations (installer only)

## Summary

This data model defines 5 key entities:

1. **Release Artifact**: Complete release package with version metadata
2. **Code Signing Certificate**: Digital certificate for Windows SmartScreen trust
3. **Update Manifest**: YAML file for auto-update discovery (latest.yml)
4. **Installation Log Entry**: Structured error logging for diagnostics
5. **Portable Settings**: Configuration for no-install portable .exe variant

All entities support the functional requirements (FR-001 through FR-015) and success criteria (SC-001 through SC-012) defined in the feature specification.
